** Key Learnings **

        1. Always add [rel="noopener noreferrer"] to external links to prevent security vulnerabilities.
        2. Separating HTML, CSS, JS into different files keeps code organized
        3. Semantic HTML improves readability and maintenance.


Part B: Ethics and Risks

    1. Stop Critical Thinking: It’s easy to stop thinking for yourself when AI gives you all the answers.

        Mitigation: I treat AI like a helpful assistant. I make sure I understand every single line of code. If I don’t understand how a piece of code works, I won’t use it in my project.

    2. Copyright Problems: AI learns from other people's work and it might give me codes belongs to someone else which has strict "do not copy" rules.
        
        Mitigation: I use AI to brainstorm ideas rather than just copying and pasting codes. I also check my code with tools and make sure I’m not using someone else's work.

    3. Environmental Impact: AI uses a massive amount of electricity and water every time we ask anything. It has a small but real "carbon footprint."

        Mitigation: I will try to be efficient with my prompts. Instead of asking the AI to rewrite the same thing multiple times, I’ll try to be clear the first time.
