    ** Key Learnings **

        1. I learned that security matters even in small details. Adding rel="noopener noreferrer" to external links prevents security vulnerabilities.

        2. I learned that CSS variables are powerful for staying organized. By defining things like spacing and colors in one central place, you spend less time fixing broken styles.

        3. I've also found that it's best practice to keep your CSS and JavaScript in their own files. It keeps the HTML codes clean.

Part B: Ethics and Risks

    ** Ethical issues and Mitigations **

        1 Losing Personal Responsibility

            It’s easy to stop thinking for yourself when an AI gives you all the answers.

                Mitigation: I treat AI like a helpful assistant, not the boss. I make sure I can explain every single line of code. If I don’t understand how a piece of code works, I won’t use it in my project.

        2. Legal and Ownership Problems

            AI learns from other people's work and it might give me codes that belongs to someone else or has strict "do not copy" rules.

                Mitigation: I use AI to help with the basic structure rather than just copying and pasting. I also use special tools to scan my code and make sure I’m not accidentally using someone else's work.

        3. Environmental Impact

            AI uses a massive amount of electricity and water so every time we ask AI something, it has a small but real "carbon footprint."

                Mitigation: I try to be efficient with my prompts. Instead of asking AI to rewrite the same thing five times, I’ll try to be clear the first time.
